Stage 0: The Basics of the Web and HTML

What a Web Page is

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

How Coding Works

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote. I'm learning to code!

Computers are Stupid

Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax").

Programmers Can't Remember Everything

There are too many details to keep everything in your head. And that's okay. If you forget how to make text italic in HTML, you can always just look it up.

Basic HTML Vocabulary

Tag: An HTML tag is always contained within angled brackets.
Element: An HTML element refers to everything within a set of opening and closing tags.
Attribute: This is a property of an HTML element.

Stage 1: Webpages, Documents, and Structure

HTML, JavaScript, and CSS

HTML represents the structure of a webpage, JavaScript is for interactive components, and CSS adds styles.

DOM (Document Object Model)

DOM is a tree-like stucture of a page, where every tage is an element in a tree.

Indentations and Boxes

It's useful to start looking at every webpage from the perspective of "seeing boxes". E.g., every header on this very page is a box, every text paragraph is a a box.

Our new language.

CSS

CSS stands for Cascading Style Sheets. The most important rule we should remember is inheritance rule: the most specific rule is applied to every element.
Here is a link if you need to reference any CSS property.

Every box consists of the following elements:

  • Margin (clears the area around the border)
  • Border
  • Padding (clears the area around the content)
  • Content (nucleus of the box; like this text inside it's "paragraph" box)

The Box Model

If you would like to make a box flexible (for better alignment of elements on a page), you need to add "display: flex" in the CSS code. Some CSS Tricks may be found here.

Code, Test, Refine

This is a routine we all should follow while designing a page:

  1. Look for natural boxes.
  2. Look for repeates styles and semantic elements.
  3. Write HTML.
  4. Apply styles (from biggest to smallest).
  5. Fix and adjust things.
